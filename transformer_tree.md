# Transformer 知识树

## 一、核心思想 — 为什么需要 Transformer？

1. **摆脱 RNN 和 CNN**
   - RNN 痛点：串行计算、无法并行、长距离依赖差
   - CNN 痛点：局部视野、需多层堆叠才能看到全局
   - → 答案：自注意力机制，一步到位看全局

2. **自注意力（Self-Attention）**
   - 每个词"看向"序列中所有其他词
   - 计算两两关联权重
   - 输出 = 所有词的加权和

3. **并行计算**
   - 不像 RNN 逐时间步处理
   - 所有位置同时计算，GPU 利用率高

## 二、模型架构

### A. 输入层
- **词嵌入**：每个 token → d_model 维向量
- **位置编码**：用 sin/cos 函数注入位置信息（因为自注意力本身不感知顺序）

### B. 编码器（Encoder）← 堆叠 N 层

**每层包含：**

1. **多头自注意力（Multi-Head Self-Attention）**
   - 公式：Attention(Q,K,V) = softmax(QK^T / √d_k) · V
   - Q/K/V 由输入线性投影得到
   - 多头：h 组独立投影，学到不同模式（语法、语义、位置…）
   - 缩放因子 √d_k：防止 softmax 进入梯度饱和区

2. **残差连接 + 层归一化**（x + Sublayer(x)）

3. **前馈神经网络（FFN）**
   - FFN(x) = max(0, xW₁+b₁)W₂ + b₂
   - 内层扩宽（d_ff=2048），外层回到 d_model

### C. 解码器（Decoder）← 堆叠 N 层

**每层包含：**

1. **掩码多头自注意力**
   - 加下三角掩码，防止看到未来词（自回归生成）

2. **交叉注意力（Cross-Attention）**
   - Q 来自解码器，K/V 来自编码器输出
   - 解码器"读取"编码器结果

3. **FFN + 残差 + 归一化**（同上）

### D. 输出层
- 线性变换 → Softmax → 概率分布 → 预测下一个 token

---

## 三、关键创新

| 创新点 | 说明 |
|--------|------|
| 自注意力 | 路径长度 O(1)，解决长距离依赖 |
| 多头注意力 | 同时关注语法、语义、指代等多种关系 |
| 位置编码 | sin/cos 函数，可外推到更长序列 |
| 完全并行 | 训练比 RNN 快几个数量级 |
| 可堆叠 | 更多层 → 更大模型 → 更强能力 |

---

## 四、复杂度对比

| 模块 | 每层复杂度 | 顺序步数 | 最大路径长度 |
|------|-----------|---------|------------|
| Self-Attention | O(n²·d) | O(1) | O(1) |
| RNN | O(n·d²) | O(n) | O(n) |
| CNN | O(k·n·d²) | O(1) | O(log_k(n)) |

---

## 五、Transformer 家族

- **编码器派**（BERT、RoBERTa）→ 理解任务
- **解码器派**（GPT、LLaMA）→ 生成任务
- **编码器-解码器派**（T5、BART）→ 序列到序列
- **跨模态**（ViT、Whisper、CLIP）→ 视觉、语音、图文

---

## 一句话总结

> **Transformer = 自注意力 + 多头 + 位置编码 + 并行堆叠**
> 用"每个词看所有词"替代 RNN 的逐步传递，实现高效并行和长程依赖建模。
