PARADIGM SHIFT
05 / 14
BEFORE / AFTER

从线性数组到树形会话

BEFORE 线性历史
msg₁ → msg₂ → msg₃ → ...
所有消息按时间顺序追加为单一数组。旧方案、失败路径和新方案混在一起，模型难以区分当前有效上下文。上下文越长，token 成本越高，裁剪又容易丢失关键决策。
  • 多路径探索互相污染
  • 无法跳转/分叉/回溯
  • 没有结构化记忆沉淀
AFTER TreeSession
Root → Branch → Leaf
会话以 JSONL 树结构存储，每条消息通过 parentId 指向父节点。当前 active branch 上的上下文完整保留，兄弟分支互不污染。支持 /jump、/fork、/clone 灵活切换。
  • 每条分支上下文独立隔离
  • 支持 /jump /fork /clone 操作
  • JSONL 可审计、可恢复

TECHNICAL BREAKTHROUGHS
09 / 14
THREE FORCES

三大技术突破

从数据结构、写入机制到检索模型，三个维度的系统性创新
01
从线性历史到树形会话事实源

将会话建模为 append-only JSONL 会话树。通过 parentId 和 activeLeafId 表达跳转、分叉、克隆等操作。模型上下文只来自当前 active branch，从结构上解决多路径探索中的上下文污染问题。

02
将压缩作为记忆提交边界

将自动记忆提交绑定到 TreeSession compaction：先形成稳定的结构化摘要，再由 SessionMemoryCommitter 写入长期记忆。这让长期记忆具有更清晰的来源、边界和可信度，避免噪声污染。

03
URI 化三层记忆 + 轻量图谱

ContextFS 将记忆保存为 URI 对象（L0摘要/L1概览/L2正文），MemoryGraph 以 JSONL 链接索引保存记忆关系，避免引入重型图数据库，兼顾图谱召回能力与系统轻量性。
